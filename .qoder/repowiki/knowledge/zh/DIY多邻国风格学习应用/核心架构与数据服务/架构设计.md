- **状态管理层**：位于 `lib/core/providers`，基于 `flutter_riverpod` 构建全局状态容器（如 `databaseProvider`, `openaiServiceProvider`），并通过 `StateNotifierProvider` 管理用户统计等复杂状态。
- **数据持久层**：位于 `lib/data`，采用 `sqflite` 实现本地 SQLite 数据库操作（`DatabaseHelper`），并定义不可变数据模型（`Deck`, `Question` 等）支持 Map 转换。
- **业务逻辑层**：位于 `lib/services`，封装外部 API 调用（`OpenAIService` 使用 `dio`）及游戏化逻辑（`GamificationService` 处理 XP、连胜等）。
- **共享 UI 层**：位于 `lib/shared/widgets`，提供遵循设计规范的原子组件（如 `DuoButton`），依赖 `lib/core/constants` 中的色彩定义。
